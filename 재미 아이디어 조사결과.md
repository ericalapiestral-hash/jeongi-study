# 재미 아이디어 60개 조사 결과 (5개 관점 브레인스토밍 + 종합 랭킹)

## ✅ 지금 구현한 것

### 1. 문 앞 한 문제 (문지기)  [✅ 구현완료]
- **무엇**: 앱을 켜면 홈보다 먼저 전체화면으로 SRS 큐에서 '오늘 가장 잊기 직전'인 문제 1개만 띄운다. 답 선택 → 3초 정오답+한 줄 해설 → 자동으로 홈. '지금 말고' 스와이프/버튼 1회로 건너뛸 수 있게 하되 하루 1회만 허용. 통과하면 홈 스트릭 옆에 '문지기 통과 ✓' 작은 배지. 기존 recordAttempt()를 그대로 호출해 SRS·통계·XP에 반영. app.js 부팅 시 renderHome 앞에 게이트 1개 삽입 = 사실상 신규 화면 1개.
- **왜**: 이 학생의 최대 실패 지점은 '앱을 열고도 뭘 할지 몰라 닫는 것'이다. 진입 자체를 인출로 바꾸면 접속=학습 1회가 무조건 보장되고, 부담이 1문제라 여는 문턱은 오히려 낮아진다. 무너진 SRS 복습률을 끌어올리는 가장 값싼 장치.
- 비용 low · 가장 싼 비용으로 '한 번 더 열게' + 인출연습을 동시에 잡는 유일한 항목이라 1위.

### 2. 틀린 이유 태그 & 약점 레이더  [✅ 구현완료]
- **무엇**: 오답 직후 해설 화면 하단에 5개 버튼: '공식을 몰랐다 / 공식은 알았는데 대입·계산 실수 / 단위·지수 실수 / 문제를 잘못 읽음 / 개념이 헷갈림'. 1탭이면 S.wrong[key].reason에 저장되고 바로 넘어감(건너뛰기 허용). 오답노트 상단에 5각형 레이더 차트 + '계산 실수 41% — 계산 도장 지수 훈련 가기' 처방 버튼(기존 dojo/codex/focus 세션으로 딥링크).
- **왜**: 기존 예상점수 통계는 '어느 과목이' 약한지만 알려주고 '왜' 틀리는지는 전혀 모른다. 20점대 학생은 개념 부족과 계산 실수가 뒤섞여 있어 처방이 안 나오는 상태다. 자유서술은 안 하지만 버튼 5개는 누른다.
- 비용 low · 기존 오답노트에 필드 하나 추가하는 수준인데, 학생이 스스로 처방을 내리게 만드는 효과가 가장 크다.

### 3. 확신도 베팅 (착각 구역)  [⬜ 다음 후보]
- **무엇**: 답을 고른 직후 채점 전에 '확실해요 / 반반 / 찍었어요' 1탭. 확실+정답=XP 2배, 확실+오답=XP 0 & 오답노트 최상단 '🔴 착각 구역'에 꽂히고 SRS 다음 복습일을 무조건 내일로 당김. '찍음+정답'은 초록이 아니라 회색 처리하고 SRS 레벨을 올리지 않음(마스터 판정에서 제외). 홈에 '착각 중인 문제 N개' 카운터. 기존 '🤷 모르겠어요' 버튼과 자연스럽게 한 세트로 통합.
- **왜**: 두 번 낙방한 사람의 20점은 '몰라서'가 아니라 '안다고 착각한 문제'에서 나온다. 지금 앱은 찍어서 맞힌 것과 확실히 아는 것을 구분하지 못해 예상점수가 부풀려진다. 이 하나로 SRS 정확도가 통째로 올라간다.
- 비용 mid · 학습효과(메타인지 교정) 1위이자, 부풀려진 예상점수라는 앱의 구조적 결함을 고치는 항목.

### 4. 세션 정서 안전장치 (첫 정답 + 3연속 오답 착지)  [✅ 구현완료]
- **무엇**: 두 규칙을 recordAttempt/세션 시작 로직에 삽입. ①세션 첫 문제는 SRS 레벨 3 이상(이미 여러 번 맞힌) 문제를 자동으로 배치하고 맞히면 '오늘 첫 정답, 감 살아있다' 한 줄. ②연속 3오답 감지 시 자동 개입: '이 유형은 원래 어렵다' + 해당 개념 핵심 한 줄(concept 재사용) + 마스터한 쉬운 문제 2개로 착지시켜 세션을 정답으로 종료하고 '오늘은 버텨낸 날'로 기록.
- **왜**: 포기는 항상 연속 오답 직후 앱을 꺼버리는 순간에 일어난다. 마지막 기억을 정답으로 덮어쓰면 다음 접속 비용이 급감한다. 신규 화면 없이 문제 선택 순서만 바꾸는 로직 변경이라 사실상 공짜.
- 비용 low · 기초 부족·2회 낙방이라는 이 학생 특성에 가장 직결되는 이탈 방지책. 코드 수십 줄.

### 5. 60점 저금통  [✅ 구현완료]
- **무엇**: 홈 상단에 통장 UI. '서로 다른 날에 3번 연속 정답'을 만족한 유닛만 '입금'으로 인정하고, 검증 유닛 수 ÷ 전체 시험 유닛 수 × 100을 '확실 점수'로 환산해 20 → 35 → 52처럼 표시한다. 기존 S.srs 데이터(레벨·마지막 정답일)만으로 계산 가능하며 새 저장 구조가 거의 필요 없다. 60 돌파 시 통장이 금색으로 바뀌고 그 날짜가 영구 기록. 옆에 '과락 위험 과목' 태그를 같이 노출.
- **왜**: 두 번 20점을 받은 학생에게 '점수'는 공포의 숫자다. 이걸 시험이 매기는 점수가 아니라 매일 쌓이는 내 자산으로 뒤집으면, 공부한 만큼 숫자가 오르는 걸 눈으로 보는 것 자체가 불안 해독제이자 재접속 이유가 된다.
- 비용 mid · 이 학생의 감정 상태를 정면으로 겨냥한 유일한 지표이고, 기존 SRS 데이터 재활용이라 비용 대비 정서 효과가 압도적.

### 6. 스트릭 방패 + 부활 시험  [✅ 구현완료]
- **무엇**: 연속 출석 3일마다 '누전차단기(방패)' 1개 적립, 최대 2개. 하루 빠지면 다음 접속 시 자동 소모되며 '차단기가 작동해 스트릭을 지켰습니다' 연출과 함께 스트릭 유지. 방패 없이 끊긴 경우엔 복귀 시 '부활 시험' 5문제(전부 마스터한 쉬운 문제)를 주고 4개 이상 맞히면 끊긴 스트릭을 그대로 복원.
- **왜**: 현재 앱엔 스트릭 복구 장치가 전혀 없어 하루 빠지면 그대로 이탈로 직행한다. 끊김을 '돌아올 이유'로 뒤집는 가장 값싼 장치이고, 마지막 시험까지 남은 기간 동안 결석 1회로 흐름이 끊기는 걸 막는다.
- 비용 low · 이탈 지점이 명확하고 대응이 검증된 패턴. 상태값 2개 추가로 끝난다.

### 7. 오답 잔량 게이지  [✅ 구현완료]
- **무엇**: 홈 최상단 고정 카운터: '아직 극복 못 한 문제 N개'. 기존 S.wrong에서 overcame=false인 항목 수를 세고, 하나 극복할 때마다 숫자가 줄며 게이지가 초록으로 차오른다. 탭하면 그 문제들만 모아 바로 푸는 세션으로 진입(기존 focus 모드 재사용). 0이 되면 축하 화면 + 배지.
- **왜**: XP는 계속 늘기만 해서 '끝'이 없지만, 줄어드는 숫자는 완결 욕구를 자극한다. 남은 3개를 없애려고 앱을 다시 켜게 만드는 구조이고, 대상이 정확히 자기 오답이라 학습효과가 100% 약점 보강으로 간다.
- 비용 low · 기존 데이터 집계 + 버튼 하나. 재방문 후크 대비 비용이 사실상 0.

### 8. 어제의 나 고스트  [⬜ 다음 후보]
- **무엇**: 세트/스테이지별로 직전 기록(문항별 소요시간·정오)을 저장하고, 같은 세트를 다시 풀 때 화면 상단에 반투명 고스트 바가 그때의 속도로 함께 진행한다. 앞서면 파란색, 뒤처지면 회색. 종료 시 '3주 전의 나를 2문제·18초 차로 이겼습니다' 판정과 달력 도장. 기록은 세트당 최고 1개만 저장.
- **왜**: 서버도 상대도 없이 만들 수 있는 유일한 진짜 대결이고, 상대가 과거의 자신이라 지든 이기든 자존감이 상하지 않는다. 실력 향상이 숫자표가 아니라 눈에 보이는 추격전으로 체감된다.
- 비용 mid · 5개 관점 중 4곳에서 중복 제안된 최다 득표 아이디어. 재도전(=반복 인출) 동기를 직접 만든다.

### 9. 버그 헌트 (숨은 실수 찾기)  [⬜ 다음 후보]
- **무엇**: 계산 도장의 정상 풀이 데이터를 코드로 변형해 한 줄에 오류를 심는다(지수 부호 뒤집기, 제곱 누락, 단위 환산 누락, 부호 반전). 학생은 '틀린 줄'을 탭해 찾고, 왜 틀렸는지 유형을 고른다. 변형은 자동 생성이라 문제를 새로 쓸 필요가 없고, 학생 본인의 '틀린 이유 태그' 상위 유형을 우선 출제한다.
- **왜**: 푸는 게 아니라 채점하는 반대 방향 사고라 같은 공식인데도 완전히 새로운 놀이처럼 느껴지고, 자기 계산 실수를 3인칭으로 객관화하게 된다. 지친 밤에도 부담이 적어 열린다.
- 비용 mid · dojo-data.js를 재활용해 신규 콘텐츠 0으로 새 모드를 하나 얻는 가성비.

### 10. 주간 성적표 카드 (저장 가능)  [⬜ 다음 후보]
- **무엇**: 일요일 접속 시 자동으로 뜨는 한 장짜리 canvas 이미지 카드: 이번 주 푼 문제 수, 새로 확정된 '저금통' 점수 변화, 최장 콤보, 극복한 오답 수, 남은 D-day, 다음 주 한 줄 목표. PNG 저장 버튼 1개. 톤은 '이번 주 이 사람은 이걸 해냈습니다'로 써서 부모님께 그대로 보여줄 수 있게 한다.
- **왜**: 이 학생의 진짜 동기는 점수가 아니라 부모님인데, 지금은 합격 통보 전까지 보여줄 증거가 하나도 없다. 중간 과정을 눈에 보이는 물증으로 만들면 그 카드를 채우려고 공부하게 되고, 주말 재접속 이유도 생긴다.
- 비용 mid · 학습효과는 간접적이지만 이 사용자에게 유일하게 작동하는 외적 동기 축이라 top에 남긴다.

## ❌ 하지 말라고 걸러진 것 (재미만 있고 공부 방해)

- **진동 채점** — 이미 fun.js의 haptic()으로 구현되어 있고 퀴즈·보스·도장에 연결됨. 새 작업이 아니라 중복.
- **오늘은 5문제** — 기존 '하루 목표(S.goal, 기본 15)' 기능과 정면 중복. 목표치 기본값을 낮추고 '여기서 꺼도 성공한 날' 문구만 추가하면 끝나는 일이라 신규 기능으로 만들 가치 없음.
- **유령 라이벌 리그 (봇 5명 주간 순위)** — 가짜 봇과의 경쟁이라는 걸 학생이 알아채는 순간 동기가 붕괴한다. 2회 낙방한 사람에게 강등·순위 압박은 불안만 키우고 학습 내용과 무관하다. 같은 효과를 '어제의 나 고스트'가 상처 없이 낸다.
- **충전 중인 배터리 (6시간 오프라인 충전)** — 학습효과가 정확히 0인 순수 리텐션 훅. '안 받으면 손해'라는 감각은 공부가 아니라 앱 여닫기를 강화하고, 마지막 시험을 앞둔 사람의 시간을 뺏는다.
- **황금 문제 (랜덤 XP 3배)** — 이미 XP·퀘스트·배지로 보상이 충분히 인플레된 상태에서 랜덤 배수를 얹으면 지표만 부풀고 학습효과는 없다. 어떤 문제를 풀지에 대한 판단도 왜곡한다.
- **승부수 사다리 (실패 시 보상 절반 손실)** — 손실을 거는 도박 구조는 두 번 떨어진 학생에게 재미가 아니라 스트레스로 작동한다. push-your-luck이 필요하면 손실 없는 '무한 등반' 최고기록형으로 충분하다.
- **부품 뽑기 & 공구함 / 내 배전반 꾸미기 / 나의 전력망 도시** — 세 개 다 같은 수집·꾸미기 축인데, 뽑기와 도감 채우기에 쓰는 시간이 문제 푸는 시간을 대체한다. 도시 버전은 신규 아트·건물 데이터까지 필요해 비용이 가장 높다. 남은 기간을 고려하면 ROI 최하위.
- **1분 릴스 문제 (무한 스크롤 피드)** — 무한 피드는 해설을 건너뛰고 스와이프하게 만들어 얕은 처리(shallow processing)를 유도한다. 문제 수만 늘고 이해는 안 남으며, 기초가 부족한 학생일수록 손해가 크다.
- **걸으며 듣기 (TTS 화면 끄기 모드)** — 듣기만 하는 수동 학습은 인출이 없어 기억에 거의 안 남고, 브라우저 TTS로 '10^-6', 'μF', '3상 Y결선' 같은 표기를 제대로 읽지 못해 오히려 잘못 외운다. 계산 과목엔 특히 부적합.
- **흔들어 복습 (DeviceMotion)** — PC(index.html 더블클릭)가 주 사용 환경이라 작동하지 않고, 폰은 HTTP 접속이라 센서 권한이 제한적이다. 우발적 팝업은 학습 흐름도 끊는다.
- **후배에게 설명하기 (파인만 모드)** — 개념 129개마다 후배 질문·설명 조각·2단계 되묻기 대본을 새로 써야 해서 비용이 이 목록 최고 수준이다. 자기설명 효과는 '해설 조각 맞추기'로 기존 텍스트 재활용해 훨씬 싸게 얻을 수 있다.
- **출동! 현장 콜 / 현장 돌발 사고** — 시나리오 10개 분기 집필 + 새 UI가 필요한 high 비용인데, 실제 필기시험 점수와의 연결은 가장 간접적이다. 합격 후 실무 감각용으로는 좋지만 지금 우선순위가 아니다.
